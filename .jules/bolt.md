## 2024-05-18 - Missing SettingsRepository->update method fixed in tests
**Learning:** Found an existing bug where `UpdateService` tried calling `$this->settingsRepo->update('system_version', $newVersion)`. The `SettingsRepository` class did not have an `update` method, it had `updateMany`. By writing unit tests, this issue was detected and successfully refactored to `$this->settingsRepo->updateMany(['system_version' => $newVersion])`. Also note that it's important not to commit random test binaries or modified vendor scripts.
**Action:** When creating tests for PHP applications, make sure not to stage or commit temporary downloaded executables like `phpunit.phar` or cache directories (`.phpunit.cache`). If platform requirements are bypassed temporarily for local test running, make sure to revert changes before committing.
