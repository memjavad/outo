## 2024-05-24 - [Hardcoded Secrets]\n**Vulnerability:** Hardcoded DB credentials and JWT secret in config\n**Learning:** Credentials were left in code instead of strictly loading from environment variables or enforcing .env existence.\n**Prevention:** Throw an error when secrets are missing instead of falling back to insecure hardcoded defaults.
## 2024-05-19 - SSRF and Path Traversal in Data Seeding Scripts
**Vulnerability:** Server-Side Request Forgery (SSRF) allowing unauthenticated fetching of arbitrary URLs, and Path Traversal allowing reading arbitrary local files in `seed_api.php` via the `json_url` parameter.
**Learning:** Developers often trust internal data seeding scripts too much, allowing dynamic input for flexibility (`http://...` or local file paths) without validating domains or sanitizing for directory traversal (`..`). Furthermore, verbose error messages revealed internal system paths when files were missing.
**Prevention:** Always validate external URL inputs against a strict allowlist (e.g., `s.nabuo.org`). Sanitize local file paths by blocking traversal sequences (`..`). Avoid reflecting raw system paths in error responses using `basename()` or generic messages.
