## 2024-05-24 - Tooltips implementation
**Learning:** Hardcoding string tooltips inside a localized app where `AppLocalizations` code is explicitly used breaks UX and testing norms if the keys aren't correctly present in `.arb` files and generated. Using existing fallback strings natively provided by Flutter or carefully using the `AppLocalizations.of(context)!.key` when properly mapped is required. Simply replacing one `IconButton`'s tooltip safely improves accessibility without risking wide-scale compilation breakage.
**Action:** Always verify `l10n` object scope and availability before batch applying translations. When making a micro-UX change, modifying a single isolated widget like `quiz_loading_state.dart` is safer and fully satisfies the single micro-UX goal.
