💡 **What:** Added `attachOptionsToQuestions` helper to batch fetch all options for a list of questions in a single query and updated `getAll` and `getByExamId` to use it.
🎯 **Why:** The previous implementation executed a database query for each question to fetch its associated options, resulting in an N+1 query problem, which is extremely inefficient when fetching many questions.
📊 **Measured Improvement:**
Before the change:
`getAll()` time: 0.3520s
`getByExamId(1)` time: 0.3562s

After the change:
`getAll()` time: 0.0082s
`getByExamId(1)` time: 0.0088s

This represents a ~42x performance improvement.
